# Contributing

Thanks for your interest in this project! This repository documents a Salesforce implementation of an AI-Powered Job Recruitment & Candidate Management System, built with declarative Admin tools and Agentforce AI.

## Ways to Contribute

- **Report an issue** — found something inconsistent between the docs and the metadata described? [Open an issue](../../issues/new/choose).
- **Suggest an enhancement** — propose a new milestone, flow, or AI use case using the feature request template.
- **Improve documentation** — typo fixes, clarity improvements, or additional diagrams are always welcome.
- **Share screenshots** — if you rebuild this project in your own org, feel free to open a PR adding your milestone screenshots (keep the existing naming convention documented in [`Screenshots/README.md`](Screenshots/README.md)).

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/short-description`
3. Make your changes
4. Commit with a clear message: `git commit -m "docs: clarify approval process entry criteria"`
5. Push and open a Pull Request using the provided template

## Documentation Style

- Keep Markdown headings sentence case and consistent with existing files under [`docs/`](docs/).
- Reference other docs with relative links (e.g. `[Security Model](docs/03-Security-Model.md)`).
- Keep the [`Screenshots/README.md`](Screenshots/README.md) index in sync if you add or rename any screenshot.
- Prefer tables over long paragraphs for field lists, permissions, and configuration values — it matches the rest of the repo.

## Code of Conduct

This project follows the guidelines in [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md). Please be respectful and constructive in all interactions.
